Kit de Pruebas de Seguridad con GUI

Este primer proyecto consiste en un “Kit de Pruebas de Seguridad” pensado para entornos controlados de laboratorio. La aplicación permite, a través de una interfaz gráfica con Tkinter, ejecutar distintos tipos de pruebas:

1. Escaneo de puertos TCP:
Utiliza la librería Scapy para construir y enviar paquetes TCP SYN. La herramienta detecta puertos abiertos, cerrados o filtrados según la respuesta (SYN-ACK, RST-ACK o ausencia de respuesta). Muestra en tiempo real cuáles de los puertos más comunes (FTP, SSH, HTTP, MySQL, etc.) están accesibles en el objetivo.

2. Pruebas de evasión de firewall:
Una vez identificados puertos abiertos o filtrados, el sistema prueba técnicas básicas de evasión: fragmentación de paquetes, uso de puertos de origen “confiables” (53, 80, 443) y TTL bajo. Cada técnica se implementa construyendo un tipo concreto de paquete TCP/IP con Scapy y analizando si la respuesta difiere del comportamiento normal, lo que indicaría reglas de filtrado débiles.

3. Crackeo de hashes (md5, sha1, sha256):
Incluye una clase “PasswordTester” que intenta descifrar contraseñas a partir de un hash. Primero realiza un ataque de diccionario, probando una lista de contraseñas comunes (y variaciones “leet” con reemplazos de caracteres). Si no se encuentra en el diccionario, ejecuta un ataque de fuerza bruta limitado (cadenas de longitud 4–6 con letras y dígitos). La interfaz solicita el hash y el tipo (por ejemplo “sha1”) y, de encontrar coincidencia, muestra la contraseña original y cuántos intentos se emplearon.

4. Prueba de credenciales SSH y de formulario web:
Para SSH se usa Paramiko: se intenta conectar con distintas contraseñas hasta encontrar la correcta o agotar el listado. Para un formulario web (“login”), se envían peticiones HTTP POST con distintas credenciales usando Requests, comparando la respuesta contra un indicador de éxito.

Tecnologías empleadas:

  • Python 3.6+
  • Scapy (paquetes de red y crudos)
  • Paramiko (autenticación SSH)
  • Requests (peticiones HTTP)
  • Tkinter (GUI: widgets, Notebook, ScrolledText, MessageBox)
  • threading (hilos para que las operaciones de red y crackeo no bloqueen la interfaz)
  • hashlib (cálculo y comparación de hashes)

Contexto y relevancia en seguridad:

  • Este kit simula, en un entorno de laboratorio, las tareas básicas de un pentester o analista de seguridad: escanear puertos, probar técnicas de evasión de firewall, crackear hashes y credenciales.
  • Escanear puertos ayuda a detectar servicios vulnerables o mal configurados en una red.
  • Comprobar evasión de firewall revela si el cortafuegos es susceptible a técnicas de fragmentación o manipulación de TTL.
  • Crackear hashes y credenciales demuestra la importancia de usar contraseñas robustas y políticas de bloqueo en SSH o formularios web con mecanismos de defensa.
  • Permite generar informes sobre puertos expuestos, reglas de firewall débiles y contraseñas vulnerables dentro de la organización.
  • Desde el punto de vista educativo, ilustra la lógica de cada técnica y cómo programar un cliente de red completo en Python.

Nivel de conocimiento requerido:

  • Python intermedio/avanzado (módulos, clases y excepciones).
  • Fundamentos de redes TCP/IP y protocolos (flags TCP, IP, TTL).
  • Manejo de librerías de seguridad en Python (Scapy, Paramiko).
  • Conceptos de hashing (md5, sha1, sha256) y ataques de fuerza bruta o diccionario.
  • Creación de aplicaciones de escritorio con Tkinter y gestión de hilos (threading).
  • Comprensión de por qué cada paso (enviar paquetes, leer respuestas, generar variaciones de contraseña, usar SSH) es relevante para evaluar la seguridad.
