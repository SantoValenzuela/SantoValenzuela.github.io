Sistema de Gestión de Contratos
con Integración a Blockchain

Este proyecto demuestra cómo llevar un problema “real” (gestión de contratos digitales) a una solución moderna y verificable, combinando una aplicación web completa con registro inmutable en blockchain. En vez de depender únicamente de una base de datos tradicional, el sistema incorpora un componente de prueba criptográfica que fortalece la trazabilidad, reduce disputas y eleva la confianza entre partes.

¿Por qué este proyecto importa?

En sistemas tradicionales, el historial de un contrato puede ser alterado (intencionalmente o por errores operativos) y la validación depende de un “tercero de confianza”. Aquí el enfoque es diferente: el contrato vive en el flujo normal de una aplicación empresarial, pero al alcanzar un estado formal (por ejemplo, cuando todas las partes aceptan), se registra en una red blockchain. Esto crea una evidencia verificable (timestamp, estado y referencia on-chain) que aporta no repudio, auditoría y transparencia.

Qué hace el sistema

  • Gestión completa del ciclo de vida: creación, edición, revisión, aceptación o rechazo y seguimiento.
  • Multiparte: invitación de participantes y control de decisiones por cada contraparte.
  • Auditoría y trazabilidad: eventos y estados del contrato visibles para inspección.
  • Registro en blockchain: al finalizar el proceso, el sistema registra el contrato mediante un smart contract, mostrando su estado on-chain.
  • API documentada: endpoints probables desde Swagger para validar comportamiento e integración.

Tecnologías emergentes aplicadas

El valor “diferenciador” de este proyecto es que no usa blockchain como adorno: la integra de forma coherente como capa de confianza. Se implementa un smart contract en Solidity (desplegado y probado en una red local con Hardhat), mientras la aplicación principal funciona como una solución empresarial moderna con SPA y API REST. Este tipo de arquitectura híbrida (Web2 + Web3) es una tendencia real en productos que requieren auditoría, cumplimiento y transparencia.

Arquitectura (pensada como producto)

  • Frontend (SPA): React + TypeScript (Vite) para una interfaz rápida, escalable y modular.
  • Backend (API REST): Node.js + TypeScript + Express como núcleo de negocio, seguridad y reglas de estado.
  • Persistencia: SQL Server para datos operativos (usuarios, contratos, participantes, eventos, etc.).
  • Capa Blockchain: smart contract en Solidity + Hardhat para registrar el contrato y su estado de forma inmutable.

Lo más retador (y lo más valioso)

El reto clave está en “hacer que ambos mundos conversen”: mantener una experiencia fluida (off-chain) sin perder la validez verificable (on-chain). Esto obliga a pensar como ingeniero de producto: manejo de estados, consistencia, confirmaciones de transacciones, seguridad en firmas y cuidado con credenciales (claves privadas / variables de entorno). Es un ejercicio completo de arquitectura moderna, no solo de código.

Casos de uso donde encaja

  • Contratos de servicios profesionales (freelance / consultoría) con trazabilidad.
  • Acuerdos empresariales internos (aprobaciones, versiones y auditoría).
  • Procesos con cumplimiento (compliance) y necesidad de evidencia verificable.
  • Plataformas B2B donde la confianza entre partes impacta directamente el negocio.
